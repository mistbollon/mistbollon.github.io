    Vue2和Vue3

    • Vue (发音为 /vjuː/，类似 view) 是一款用于构建用户界面的 JavaScript 框架。它基于标准 HTML、CSS 和 JavaScript 构建，并提供了一套声明式的、组件化的编程模型，帮助你高效地开发用户界面。无论是简单还是复杂的界面，Vue 都可以胜任。

    Vue3 的好处

    • Vue3 运行性能大幅提升，速度是 Vue2 的 1.5 倍左右

    • Vue3 支持 tree shaking，可以进行按需编译，编译后的文件体积比 Vue2 更小

    • Vue3 组合式 API 使应用中的功能代码更聚合，使组件间公共逻辑的抽取更容易

    • Vue3 中提供了更加先进的功能，比如 teleport，suspense 等

    • Vue3 对 TypeScript 的支持更加友好，对大型前端应用的支持更加游刃有余

    • Vue 是目前国内前端使用者最多的框架，Vue 官方已经将 Vue3 作为默认版本使用

    Vue2 和 Vue3 区别

    阶段生命周期
    Vue2 Vue3 执行阶段
    beforeCreate beforeCreate(setup()) 组件初始化阶段
    created created(setup()) 组件初始化完成
    beforeMount beforeMount 组件挂载之前
    mounted mounted 组件挂载完成
    beforeUpdate beforeUpdate 数据更新，DOM 完成更新之前
    updated updated 数据更新，DOM 完成更新
    beforeDestroy beforeUnMount 组件销毁之前
    destroyed unmounted 组件销毁完成
    activeted onActivated 被缓存组件再次渲染时触发
    deactivated onDeactivated 被缓存组件销毁时触发
    • 如果组件被<keep-alive>包含，则会有 onActivated 阶段和 onDeactivated 阶段。
    • <keep-alive>是 vue 的一个内置组件，能够实现组件缓存，使被包含的组件保存状态，避免重复渲染。
    组件通信

    vue2使用的是选项式API（Options API），响应式原理是利用的 es5 的 API，通过 Object.defineProperty()对数据进行劫持结合发布订阅模式来实现。

    vue3使用的是组合式API（Composition API），使用 es6 的 proxy API 对数据进行代理。

    Proxy API 对象的 Proxy 对象用于定义基本操作的自定义行为（属性查找、赋值、枚举、函数调用等）proxy 对象时目标对象的一个代理器，对任何目标对象的操作，都必须通过该代理器。可以对外界的操作进行拦截、过滤或修改等操作。

    const p = new Proxy(target(目标对象), handler(属性对象))
